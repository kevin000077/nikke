# 弹珠轨迹助手

这是一个 Windows 10/11 桌面辅助工具：它只读取指定游戏窗口的画面，在窗口上方绘制透明、鼠标穿透的预测轨迹，不读取游戏内存，也不会控制鼠标或自动发射。

## 安装

要求 64 位 Python 3.12。建议在本目录使用独立虚拟环境：

```powershell
python -m venv .venv
.\.venv\Scripts\python.exe -m pip install --upgrade pip
.\.venv\Scripts\python.exe -m pip install -e ".[test]"
```

## 启动

可以直接启动并从列表选择游戏窗口：

```powershell
.\.venv\Scripts\python.exe -m marble_aim
```

分享版 EXE 会保留控制台窗口，用于显示启动进度和完整错误信息。首次选窗对话框会强制置顶；如果已经记住旧窗口，也可以点击“选择窗口”重新选择。

也可以指定标题关键字：

```powershell
.\.venv\Scripts\python.exe -m marble_aim --window-title "游戏标题"
```

首次使用建议打开调试框并执行校准：

```powershell
.\.venv\Scripts\python.exe -m marble_aim --window-title "游戏标题" --debug-view --calibrate
```

## 热键与颜色

- `F7`：显示或隐藏覆盖层。
- `F8`：暂停或继续识别。
- `F9`：开始 20 秒校准；期间完成 3–5 次向左、中、右的发射。
- `F10`：调整弹珠半径、反射角、发射点和调试框。
- `重启识别`：完整重新识别外框、方块、球半径和发射点。
- `设置白框`：依次点击左右竖线、上下横线，手动构造碰撞框。
- `选择窗口`：随时切换要捕获的游戏窗口。
- `退出助手`：保存配置、释放截屏资源并结束后台进程。
- 亮紫色线：当前瞄准方向及其后续预测轨迹。
- 绿色虚框：本回合已经锁定、实际参与碰撞计算的方块盒。

配置默认保存在 `%LOCALAPPDATA%\MarbleAim\config.json`。仓库中的 `config.example.json` 包含所有字段。

## 识别与校准说明

识别器优先检测水池中浅色碰撞框的四条内沿，再在框内按“高饱和非水色区域 + 规则网格”识别方块，因此支持绿色、黄色、紫色等方块，也能拆分紧挨在一起的方块。若画面没有可检测的白框，才回退到 `config.json` 中的 `water_hsv_*` 和 `block_hsv_*`。

自动白框识别不稳定时，可点击“设置白框”。鼠标会依次跟踪左竖线、右竖线、上横线和下横线，每次左键点击确定一条边；右键或 `Esc` 取消。完成后坐标按窗口比例保存，之后直接使用手动框并跳过自动白框识别。点击“重启识别”会清除手动框并恢复完整自动识别。

游戏自带的白色发射虚线会通过长直线投票识别；识别成功时，当前预测轨迹采用游戏虚线方向实时刷新，虚线未出现时隐藏轨迹与推荐，避免飞行阶段误识别。

数字 OCR 已移除，不再参与识别、场景锁定或轨迹计算。两条自动推荐路线当前已停用，只显示手控瞄准方向的单条预测路线。

程序按回合工作：初次瞄准线出现时识别并冻结白框和方块格位；瞄准期间使用独立快速检测和数学碰撞模型约每 33ms 刷新轨迹。发射点由当前虚线与白框底边的交点反推，并经过短窗口中值稳定。底部发射箭头是硬门控：箭头不存在时，飞行弹珠、白色特效和残影不能产生预测线。瞄准线连续消失 600ms 后判定小球飞行；飞行后瞄准线连续出现 600ms，程序短暂隐藏覆盖层并用干净画面比较方块绝对网格。方块发生变化时只更新方块和发射点，初始白框保持固定。

校准使用画面运动差分追踪弹珠，并从墙壁转向点拟合边界偏移与反射修正。校准时尽量避免打开菜单或移动游戏窗口。若提示反弹样本不足，重新按 `F9` 并制造至少两个清晰的左右墙反弹。

`--debug-view` 会在覆盖层显示识别框，并每秒更新 `debug/latest_detection.png`。调试结束后可不带此参数启动。

碰撞模型以弹珠中心为轨迹点：上、左、右边界会按弹珠半径向内缩，确保弹珠外缘接触边界时反弹；底边是开放出口，弹珠向下穿出后立即终止轨迹，不会反弹。

覆盖层会在每个反射转折点绘制一个与模型半径相同的半透明球，可直接检查球外缘是否贴住墙面或方块。`F9` 校准功能仍保留，校准结果继续写入球半径、边界偏移和反射角修正参数。

球半径不使用固定屏幕像素：优先从当前瞄准圈测量，并用最近可靠的“半径/白框宽度”比例稳定；测量失败时按方块边长 `0.215` 或白框宽度 `0.0345` 回退。窗口缩放后同步变化。`F9` 校准得到的新半径会反写为当前分辨率下的方块比例。

## 当前边界

- 方块主体按直角正方形处理；下方深色底座和投影不计入碰撞盒。弹珠仍按圆形处理，因此球心绕过正方形尖角时会自然形成一个球半径的圆弧。
- 多颗弹珠默认共用同一初始方向；推荐分数按单条几何轨迹的方块碰撞次数计算。
- 窗口必须可见且不能最小化。
- 如果游戏使用受保护或独占全屏渲染，普通屏幕捕获可能得到黑屏；请切换窗口化或无边框窗口模式。

## 录屏预测对比

确定松手前最后一帧后，可把这一角度的单条预测路径以紫色虚线叠加到录屏：

```powershell
.\.venv\Scripts\python.exe tools\overlay_video_prediction.py `
  "input.mp4" "prediction.mp4" `
  --aim-frame 1797 --start-time 64 --end-time 77 `
  --ball-radius 7 --max-collisions 12
```

脚本会用末帧之前的多帧中位数稳定角度，并从发射线反推人物横向发射位置，同时生成同名 `.json` 参数记录和 `.preview.png` 预览图。
